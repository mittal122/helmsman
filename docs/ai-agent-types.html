AI Agent Types — and a practical use case for each.

A survey of the main categories of AI agents, what each one does, and a real-world problem it solves. The final row marks the type selected to build for this project.

★

Selected to build: Autonomous DevOps / Deployment Agent Our pick

Codename Helmsman — an AI agent that takes a developer's finished app and gets it running live on Kubernetes (modern cloud infrastructure): it prepares everything, checks it, deploys it, returns a working link, then keeps monitoring and auto-fixing problems — all in full view. It solves a painful, high-value, everyday problem, and has a clear, demonstrable success condition: the app goes live and stays healthy.

Why this pick

Why the DevOps / Deployment Agent is the right one to build

  • Solves a real, painful problem. Deploying to Kubernetes normally needs a scarce specialist — most developers get stuck at the last step.
  • Clear, demonstrable success. Either the app is live and healthy, or it isn't. Easy to show working, easy to measure.
  • High business value. It removes a bottleneck, reduces reliance on rare experts, and speeds up every release.
  • Naturally fits an agent design. It combines several agent ideas above — a coordinator, monitoring, and problem-diagnosis — into one practical product.
  • Safe to scope. It can start small (deploy one app locally) and grow in clear phases, so there is something working early.
