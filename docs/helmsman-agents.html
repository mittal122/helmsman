Project: Helmsman  ·  AI Deployment Agent

The agents inside Helmsman — and what each one does.

Helmsman isn't one big program. It's built as a team of specialist agents, each responsible for one part of getting an app deployed and kept healthy — coordinated by a single lead. Here is the full roster, with a practical example of what each one does.

The roster

9 agents, each with a practical use case

Agent What it does Practical use case (example) Kind
Coordinator Runs the whole deployment, decides what happens next, and always knows the current stage, health, and progress. User starts a deployment — the coordinator sequences every step and keeps the dashboard showing exactly where things are. Lead
Onboarding Agent Helps package an app that isn't ready for deployment yet. The user's app has no container setup — the agent writes a ready-to-use prompt they paste into their own AI assistant to create it. AI
Config-Advisor Agent Suggests sensible deployment settings and explains each one in plain language. The user doesn't know how many copies to run or how much memory to give — the agent proposes safe values and explains why. AI
Error-Resolution Agent Diagnoses failures, explains them simply, and recommends a fix. The app runs out of memory and crashes — the agent says “it needs more memory” and recommends the exact change. AI
Manifest-Generation Agent Creates all the Kubernetes setup files from the collected details. Turns the user's answers into the complete set of deployment files — no hand-writing required. Automated
Validation Agent Checks every file for problems before anything is deployed. Catches a misconfigured port or missing value before it ever reaches the live cluster. Automated
Deployment Agent Deploys the app and watches it come online. Applies the files and reports progress live — “2 of 2 copies ready” — then confirms the app is healthy. Automated
Monitoring (SRE) Agent Continuously watches health, logs, and resource usage after the app is live. Notices a copy of the app stuck in a restart loop and immediately flags it. Automated
Rollback Agent Restores the last working version when a deployment goes wrong. A bad update fails to start — the agent automatically reverts to the previous stable version. Automated
Lead — coordinates the others AI — reasons & explains in plain language Automated — precise, repeatable actions
How they work together

One lead, many specialists — and a deliberate use of AI.

Every agent reports through the coordinator, which always knows the current stage, health, and any errors — so the whole system moves as one and nothing happens out of order.

Why only three use AI: AI is applied exactly where real judgement or plain-language explanation is needed — guiding the user and understanding failures. The agents that actually touch the live system are precise, automated specialists built on proven industry-standard tools. This keeps Helmsman predictable, safe, and trustworthy: the AI advises, it never improvises with live systems.
